Point Example for C++

  • Map.idl

    • IDL defenitions for the Point and ConnectedPoint value types, boxed string valuetype Label, and the PointManager
    interface.
  • Server.C

    • The main server implementation file. Creates a POA named serverPoa, creates an instance of PointManager servant and activates  that servant on the serverPoa. It then starts waiting for client requests. This file also implements  the PointManager servant. The operations inolve ,creating either a derived point ( ConnectedPoint ) or a plain point (Point) , getting the point  information, listing the points created,  connecting the point( only if it is a ConnectedPoint ) to a set of points and destroying a point. Each operation refers to a point with a value box String Label which is provided when it is created. The implementation throws exception that can be caught in the client for error inputs.
  • Client.C

    • This file contains the main Client implementation . The client starts by displaying a command prompt .The user can enter
    a set of commands ( see below ) that enable the creation or other operation to be invoked. You may notice that the
    value factories for the valuetypes Point and ConnectedPoint are registered with the orb. This is essential for the orb to
    unmarshal the valuetype from 'on the wire' representation. 
  • PntImpl.h

    • This file contains the Point and ConnectedPoint  implementation . 
  • Makefile.cpp

    • Used to build all the targets . 

Building this example

Typing make -f Makefile.cpp (or nmake -f Makefile.cpp on Windows) in the point subdirectory will cause the Map.idl to be run through the idl2cpp compiler. It will also build the following executables for the examples described above:
  • Server (Server.exe on Windows)
  • Client (Client.exe on Windows)

Running this example

To run the examples, first make sure that the VisiBroker Smart Agent (osagent executable) is running on your network. Then start the bank server using the command: 
    prompt> Server  & 
(start Server on Windows)
// make the server run in the background
    prompt> Client

    You will see a prompt
     Command-->

    enter the commands based on the following menu

    createbase <label>
    createderived <label>
    connect <label>
    destroy <label>
    get <label>
    list <num copies>
    help ( or ? )
    quit
