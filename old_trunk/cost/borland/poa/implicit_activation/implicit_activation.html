Implicit Activation Bank

The example has the standard AccountManager and Account interfaces.  AccountManager is used for opening an account and Account is used for querying the balance on the account.  The importance of the example is to show how to set POA policies such that servants can be automatically activated by the POA whenever a reference or id associated to the servant is obtained.  The reference obtained is published to the client via a file "ior.dat".

Both the servants for AccountManager and Account are activated by myPOA using implicit activation.


From this example, you will learn how to:
  • To use the right policies on POA creation so it can activate servants implicitly
