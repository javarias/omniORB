Multiple ID Bank

The example has the standard AccountManager and Account interfaces.  AccountManager is used for opening an account and Account is used for querying the balance on the account.  This example creates two object references, one for Checking and another for Savings AccountManagers and writes it out to files "cref.dat" and "sref.dat" respectively and starts waiting for requests. It also sets up a default servant on the POA. All the requests coming on the POA go through the default servant.

The MULTIPLE_ID IdUniqueness policy is used to create the POA.  With this policy, our default servant can service more than one object.

Since the same default servant handles all requests on the POA, that servant may require some information on the object id. It can obtain that information through _object_id() method.


From this example, you will learn how to:
  • Use the right policies on POA creation so it can use default servant
  • Use the right policies on POA creation so the default servant and service multiple objects
  • Set the default servant on the POA
  • Create a reference with no servant backing up that reference
  • Use the _object_id() method in the default servant implementation to distinguish requests.
