Server Engine Policy Bank

The example has the standard AccountManager and Account interfaces.  AccountManager is used for opening an account and Account is used for querying the balance in the account. The importance of this example is to show how to create your own server engine and how to form the corresponding server engine policy and associate the POA with that policy.  All requests made to objects on the POA will conform to the server properties listed by the server engine.  This example creates an object reference for the AccountManager and writes it out to the file "ior.dat" and starts waiting for requests.


From this example, you will learn how to:
  • Get the property manager from the orb.
  • Add properties using the property manager.
  • Create a server engine policy.
  • Set the different attributes of the server engine.
