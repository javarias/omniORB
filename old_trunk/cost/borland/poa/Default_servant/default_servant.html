Default Servant Bank

The example has two simple interfaces - SavingsAccountManager or CheckingAccountManager to open an Account and to query the balance in that account. This example creates two object references, one for Checking and another for Savings AccountManagers and writes it out to files "cref.dat" and "sref.dat" and starts waiting for requests. It also sets up a default servant on the POA. All the requests coming on the POA go through the default servant.

Since the same default servant handles all requests on the POA, that servant may require some information on the object id. It can obtain that information through _object_id() method.

Default servant is ideally suited for serving a large number of servants when all the servant state is stored in a database. The incoming object id can be used as a key in to the database to retrieve the state.


From this example, you will learn how to:
  • To use the right policies on POA creation so it can use default servant
  • To set the default servant on the POA
  • To create a reference with no servant backing up that reference
  • To use the _object_id() method in the default servant implementation to distinguish requests.
