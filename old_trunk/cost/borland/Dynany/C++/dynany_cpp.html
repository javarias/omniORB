DynAny Example for C++

The Client application program will illustrate how to create and initialize a DynStruct type.  The Client will then create an any type from the DynStruct which can then be passed to the Server application program for evaluation.  The Server application will illustrate how to traverse, extract, and display the primitive constituents of the data value associated with an any at run-time.   It should be pointed out that the Server application was written such that it does not have any static information for the type of the any.

Directory Contents

  • Printer.idl

    • IDL interface for the PrinterManager object and associated type values.
  • Server.C

    • PrinterManager server. Creates an instance of the PrinterManager and activates it  to make this object available to client programs.  The PrinterManager Server implementation( printerManagerImpl)  implements two methods: printAny, which displays the StructType info contained within a dynamic any type, and shutdown, which deactivates the PrinterManager instance and then shuts down the PrinterManager server process.
  • Client.C

    • This is the PrinterManager client. It binds to the PrinterManager object and invokes printAny() by passing an any value obtained from a Created DynStruct  which contains the struct StructType. The DynStruct is initialized by creating a Name value pair of struct member names and values .The client then invokes shutdown() on the PrinterManager object which will then unregister and deactivate the PrinterManager object and shutdown the PrinterManager Server.
  • Makefile.cpp     (vbmake.bat on Windows): Used to build all the test targets.

Building this example

Typing make (or vbmake on Windows) in the DynAny subdirectory will cause the Printer.idl  to be run through the idl2Cpp compiler. It will also build the following executables:
  • Server.exe
  • Client.exe

Running this example

To run the examples, first make sure that the VisiBroker Smart Agent (osagent executable) is running on your network. Then start the PrinterManager Server using the command: 
    prompt> Server  & 
// make the server run in the background
Next, Observe the server display the structure member information and values by typing: 
    prompt> Client
