Interface Repository Example

Files in this example

  • Bank.idl: The definition of Account and AccountManager interfaces
  • AccountManagerImpl.h: The servant that implements the AccountManager interface
  • AccountImpl.h: The servant  that implements the Account interface
  • Server.C: The server mainline
  • Client.C: The client mainline

Building this example

Typing make cpp (or nmake cpp on Windows) in the ir subdirectory will this example to be built. This will include the following steps

1. Compile the Bank.idl and generate all CORBA supporting classes
2. Compile the generated stubs and servants, the servant implementations for interfaces described in Bank.idl
3. Compile the Client and Server mainlines.

Running this example

To run the examples, first make sure that the VisiBroker Smart Agent (osagent executable) is running on your network.

Start the Interface Repository using the command: 

prompt> irep myIrep Bank.idl  &
start irep myIrep Bank.idl (on Windows)
// make the irep run in the background
// you can use any name for the Interface Repository instead of myIrep

Then start the bank server using the command: 

prompt> Server  & 
start Server (on Windows)
// make the server run in the background

Next, open some users'  bank account with initial balances 

prompt> Client peter 12.213

or 

prompt> Client peter  
// uses a random initial peter's account balance value

or 

prompt> Client   
// uses a default name and random balance value

The example server first creates in the Interface Repository an additional operation definition in the Account interface as if it would described in there by the user. The server's implementation already has the DSI code to handle for this operation to proceed. The example client first obtains a reference to the balance object with name specified. If it had not been yet created, the AccountManager creates one and initializes its balance to a random value. The client obtains this current balance and prints it on the screen. Then client modifies the balance value either with one specified by the user or with the randomly taken one. And, finally, client verifies with the Interface Repository if this Account interface supports the describe operation. If it is the client obtains the Account's description via DII and prints it on the screen.
